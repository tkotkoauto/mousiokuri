# 申し送りWebアプリ 仕様書

## 概要

在宅訪問クリニック向けの申し送り（カンファレンス用振り返り入力）Webアプリ。
純粋なHTML/CSS/JS（シングルファイル）で実装。データはブラウザのメモリのみに保持し、クラウドへの送信・保存は一切行わない。
院内でURLを共有して使用。GitHub Pagesでデプロイ。

---

## 技術スタック

| 項目 | 内容 |
|------|------|
| 実装形式 | 単一HTMLファイル（`index.html`） |
| スタイル | CSS（Tailwind CDN or インラインCSS） |
| ロジック | バニラJavaScript（フレームワークなし） |
| データ保存 | なし（セッション内メモリのみ） |
| デプロイ | GitHub Pages（`gh-pages`ブランチ or `main`の`/docs`） |
| 院内共有 | GitHub PagesのURL |

---

## 画面フロー

```
[1] 申し送り者選択
        ↓
[2] 患者リスト画面
    - 患者名入力（最大10名）
    - 「追加」ボタンで患者追加
    - 各患者の「inclde基準」チェックボックス
    - チェックが1つ以上 → 振り返り入力へ
        ↓（チェックあり患者ごとに繰り返し）
[3] 振り返り入力画面（患者ごと）
    - チェックされた基準に応じたフォームセクション表示
    - 入力完了 → 次の患者へ / 全患者完了 → サマリーへ
        ↓
[4] サマリー・コピー画面
    - 全入力内容をフォーマットして表示
    - 「コピー」ボタン → クリップボードへコピー
```

---

## [1] 申し送り者選択画面

- 事前登録済みの主治医名リストからトグル（ラジオボタン or ボタン選択）で1名選択
- 選択後は以降の画面には申し送り者名を表示するのみ（トグルは非表示）
- 主治医リスト（初期値、HTMLに直書き・後から編集可能）：
  - 例：田中先生、鈴木先生、佐藤先生 など（実装時にクリニックの先生方に合わせて設定）

---

## [2] 患者リスト・Include基準チェック画面

### 患者入力
- 患者名入力フィールド + 「追加」ボタン
- 患者カード形式で一覧表示（最大10名程度を想定）

### Include基準チェックボックス（患者ごと）

| # | 基準 | 略称（コピー時の見出し） |
|---|------|------------------------|
| 1 | 体調に変化、処方の変化があった症例 | 体調・処方変化 |
| 2 | もやもやした感情が湧いた症例 | もやもや |
| 3 | 新規受け入れ・退院直後 | 新規・退院直後 |
| 4 | 救急搬送・入院になった症例 | 救急・入院 |
| 5 | 看取り・死亡 | 看取り・死亡 |
| 6 | ACP/方針転換があった | ACP・方針転換 |
| 7 | 家族・介護者の問題 | 家族・介護者問題 |
| 8 | 多職種から相談があった案件 | 多職種連携 |
| 9 | 検査結果で要対応のもの | 検査要対応 |
| 10 | ヒヤリハット・インシデント | ヒヤリハット |
| 11 | いつもと違う判断をした症例 | 臨床判断の変化 |

- チェックが1つもない患者 → 申し送り対象外（スキップ）
- 「振り返りを開始」ボタンでチェックあり患者の入力フローへ

## [3] 振り返り入力画面（患者ごと）

### セクション表示条件（チェックされた基準に応じて動的表示）

| セクション | 表示トリガーとなるInclude基準 |
|-----------|------------------------------|
| **判断の根拠** | 1, 4, 6, 8, 9, 11 |
| **もやもや・疑問** | 2, 6, 7 |
| **Take Home（学びログ）** | 5, 11（任意：全症例に表示も可） |
| **1例振り返り** | 3, 5（1例深堀りとして選択式） |
| **SEA（短縮版）** | 4, 10 |
| **タグ** | 常に表示 |

> **議論ポイント①**: 複数のInclude基準がチェックされた場合、該当セクションをすべて表示するか、代表1つに絞るか？
> → 現状案：すべて表示（入力は任意、空欄スキップ可）

### 各フォームセクションの入力フィールド

#### 判断の根拠
- なぜその方針にしたか（テキストエリア）
- 採らなかった選択肢と理由（テキストエリア）
- 参考にしたガイドライン・文献（テキスト、任意）

#### もやもや・疑問
- 引っかかったこと（テキストエリア）
- なぜ引っかかったかの仮説（テキストエリア）
- 確認したい/学びたい（テキストエリア）

#### Take Home（学びログ）
- 今回の自分への一言（テキスト）
- 次に同じ場面で試したいこと（テキストエリア）

#### 1例振り返り
- なぜこの1例を選んだか（1行テキスト）
- 振り返り内容（テキストエリア）

#### SEA（短縮版）
- 何が起きた（テキスト）
- なぜ（主要因1つ）（テキスト）
- 学び（1つに絞る）（テキスト）
- 次やること（SMARTで1つ）（テキスト）

#### タグ
- 疾患タグ（テキスト、例: #糖尿病 #心不全）
- 学びテーマタグ（テキスト、例: #判断 #家族対応）

### ナビゲーション
- 「この患者の入力を完了」ボタン → 次の対象患者へ
- 最終患者完了 → サマリー画面へ
- 進捗表示（例: 患者 2/5）

- 引っかかったこと：〜〜〜
- 仮説：〜〜〜
- 確認したいこと：〜〜〜

#疾患: 〜〜  #学びテーマ: 〜〜

### コピー機能
- 「クリップボードにコピー」ボタン
- コピー成功時：「コピーしました！」フィードバック表示（2秒）
- コピー後はメッセージアプリ（LINE/Slack等）に貼り付けて使用

## デプロイ方法

```bash
# GitHubリポジトリ作成・初期化
git init
git add index.html
git commit -m "initial"
git remote add origin https://github.com/<org>/申し送りapp.git
git push -u origin main

# GitHub Pages設定
# Settings → Pages → Source: main branch / root
# → https://<org>.github.io/申し送りapp/ でアクセス可能
```

- **プライベートリポジトリ + GitHub Pages**はGitHub Freeでは非公開ページ不可（要有料プランまたは組織アカウント）
- 代替案：パブリックリポジトリ（URLを知っている人のみアクセス、入力データはブラウザのみ）

> **議論ポイント②**: GitHubリポジトリはパブリック/プライベートどちらにするか？（院内共有のみであればパブリックでもURLさえ知らなければ問題は少ないが、確認必要）

---

## 確定事項

1. **セクション表示**: 複数criteria → 該当セクションすべて表示（空欄スキップ可）
2. **リポジトリ可視性**: パブリック（データはブラウザのみ・クラウド保存なし）
3. **主治医リスト**: 姜・熊井・土屋・西山・川村・成瀬・平塚・山中・服部・星長・谷本・森家・石井・大野
4. **1例振り返り**: 全症例に任意表示
5. **Take Home**: 全症例に任意表示
6. **患者名**: フルネーム入力可（データ保存なし）
