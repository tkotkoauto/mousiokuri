# 申し送りWebアプリ 仕様書

## 概要

在宅訪問クリニック向けの申し送り（カンファレンス用振り返り入力）Webアプリ。
純粋なHTML/CSS/JS（シングルファイル）で実装。データはブラウザのメモリのみに保持し、クラウドへの送信・保存は一切行わない。
院内でURLを共有して使用。GitHub Pagesでデプロイ。

---

## 技術スタック

| 項目 | 内容 |
|------|------|
| 実装形式 | 単一HTMLファイル（`index.html`） |
| スタイル | インラインCSS（バニラ） |
| ロジック | バニラJavaScript（フレームワークなし） |
| データ保存 | なし（セッション内メモリのみ） |
| デプロイ | GitHub Pages（masterブランチ / root） |
| 院内共有URL | https://tkotkoauto.github.io/mousiokuri/ |
| GitHubリポジトリ | https://github.com/tkotkoauto/mousiokuri |

## 画面フロー

```
[1] 申し送り者選択
        ↓
[2] 患者リスト画面
    - 患者名入力（最大15名）
    - include基準の一覧を参照表示（オレンジカード内）
    - 各患者カードでinclude基準チェックボックス
    - 1つ以上該当した患者が申し送り対象
        ↓（対象患者ごとに繰り返し）
[3] 振り返り入力画面（患者ごと）
    - プログレスバーで進捗表示（患者 n/m）
    - チェックされた基準に応じたフォームセクション動的表示
    - 入力完了 → 次の患者へ / 全患者完了 → サマリーへ
        ↓
[4] サマリー・コピー画面
    - 全入力内容をフォーマットして表示
    - 「コピー」ボタン → クリップボードへコピー
    - コピー後はメッセージアプリに貼り付けて使用
```

---

## [1] 申し送り者選択画面

- 登録済み主治医をグリッドボタンで表示、1名を選択
- 選択後は以降の画面では名前を表示するのみ（ボタンは非表示）
- 主治医リスト：姜・熊井・土屋・西山・川村・成瀬・平塚・山中・服部・星長・谷本・森家・石井・大野

---

## [2] 患者リスト・include基準チェック画面

### include基準（参照用）
画面上部のオレンジカード内にinclude基準11項目を常時表示（チェック時の参考として）。

### 患者入力
- 患者名入力フィールド＋「追加」ボタン（Enterキーでも追加可）
- 患者カード形式で一覧表示（最大15名）
- チェックが入ると患者カードの左ボーダーが緑に変わる（視覚的フィードバック）

### include基準チェックボックス（患者ごと）

| # | 基準 | 略称（コピー時） |
|---|------|----------------|
| 1 | 体調に変化、処方の変化があった症例 | 体調・処方変化 |
| 2 | もやもやした感情が湧いた症例 | もやもや |
| 3 | 新規受け入れ・退院直後 | 新規・退院直後 |
| 4 | 救急搬送・入院になった症例 | 救急・入院 |
| 5 | 看取り・死亡 | 看取り・死亡 |
| 6 | ACP/方針転換があった | ACP・方針転換 |
| 7 | 家族・介護者の問題 | 家族・介護者問題 |
| 8 | 多職種から相談があった案件 | 多職種連携 |
| 9 | 検査結果で要対応のもの | 検査要対応 |
| 10 | ヒヤリハット・インシデント | ヒヤリハット |
| 11 | いつもと違う判断をした症例 | 臨床判断の変化 |

- チェックが1つもない患者 → 申し送り対象外（スキップ）

## [3] 振り返り入力画面（患者ごと）

### セクション表示ルール

| セクション | 表示条件 |
|-----------|---------|
| **判断の根拠** | include基準 1,4,6,8,9,11 のいずれかをチェック |
| **もやもや・疑問** | include基準 2,6,7 のいずれかをチェック |
| **SEA（短縮版）** | 全患者に任意表示 |
| **Take Home（学びログ）** | 全患者に任意表示 |
| **1例振り返り** | 全患者に任意表示。サブタイトル「本日の患者の中で1例深堀り」 |
| **タグ** | 全患者に常時表示 |

### 各フォームセクションの入力フィールド

#### 判断の根拠
- なぜその方針にしたか（テキストエリア）
- 採らなかった選択肢と理由（テキストエリア）
- 参考にしたガイドライン・文献（テキスト、任意）

#### もやもや・疑問
- 引っかかったこと（テキストエリア）
- なぜ引っかかったかの仮説（テキストエリア）
- 確認したい/学びたい（テキストエリア）

#### SEA（短縮版）
- 何が起きた（テキスト）
- なぜ（主要因1つ）（テキスト）
- 学び（1つに絞る）（テキスト）
- 次やること（SMARTで1つ）（テキスト）

#### Take Home（学びログ）
- 今回の自分への一言（テキスト）
- 次に同じ場面で試したいこと（テキストエリア）

#### 1例振り返り
- サブタイトル：本日の患者の中で1例深堀り
- なぜこの1例を選んだか（1行テキスト）
- 振り返り内容（テキストエリア）

#### タグ
- 疾患タグ（テキスト）
- 学びテーマタグ（テキスト）

- 引っかかったこと：〜〜〜
